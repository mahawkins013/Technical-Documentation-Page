Introduction to HTML and CSS

HTML and CSS allow you to build your own webpages! HTML code gives you the basic input and some basic interactive content of your page, while CSS allows you to style that content and really bring your page to life with color, positioning, interactivity, and more.

While HTML and CSS are compatible with all browsers, some older versions of browsers will not accept newer features of HTML5 and CSS3, such as HTML5's 'Flexbox', or CSS3's Grid function.

This page will serve as a basic introduction to HTML and CSS concepts and allow you to start building your first webpage, but it is by no means comprehensive.

Introduction to HTML Elements

Many elements require closing tags, but some do not. When an element does not require a closing tag, you can either proceed with brackets as normal on an opening tag, or you can include a space and a forward slash before the bracket at the end of the element to indicate that no closing tag is needed. For example, the meta element tag, which provides metadata information for your page can be written in either of these two ways:

<meta charset="UTF-8">
or
<meta charset="UTF-8" />
are acceptable ways of writing the <meta> element

Below are some other common HTML tags that do not require closing tags:

<link />
<img />
<br>

Many elements, however, do require a closing tag, otherwise you will experience errors in your code as following elements will be incorrectly read as the element without the required closing tag.

Here are some basic elements that require closing tags:

<html></html>
<head></head>
<body></body>
<figure></figure>
Basic HTML Boilerplate

All html websites should contain a basic boilerplate allowing them to be opened as a webpage, and allowing your CSS stylesheet to be read.

Each document must start with this declaration to let the browser know it is an html document (without the spaces between the brackets):

<!DOCTYPE html>

After your html doctype declaration, you need your html and then your head element to start adding the rest of your boilerplate:

<html>
<head>
</head>
</html>

All of your elements will go within the < html > tag. Your first addition to the < head > element will be the title of your page. This will not show up on your page but will provide information for search engine optimization and will show up in the tab for your page as well as in the URL:

<title>Introduction to HTML and CSS</title>

Within your head tag, you need to add these meta tags:

This charset tag specifies the character encoding for your document. 'UTF-8' covers all unicode characters (and supports many languages):

<meta charset="UTF-8" />

This charset tag ensures that your page will fit the screen size of the device it is being viewed on, and that it will set the initial zoom to fill the screen:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This link element will allow you to link your CSS document and add some styling to your page (please note, you can name your css document anything and input this name in place of 'styles.css' below as long as you save it as a css document with .css. Examples: blog.css, styling.css, fun.css):

<link href="styles.css" rel="stylesheet" type="text/css" />

Your resulting basic HTML boilerplate should look like this:

<!DOCTYPE html>
<html>
<head>
<title>Your Title Here</title>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<link rel="stylesheet" href="styles.css" type="text/css" />
</head>
</html>
Interactive HTML Elements

There are many HTML elements that will allow viewers of your page to interact with your site - all without using any CSS!

Adding a <textarea> element to you form will allow users to input text. Embedding your textarea within a form element allows you to see a user's input when the form is submitted to the location of your choice. Within the form element, you can either use the 'action' attribute to send form results to a web address, or you can use the 'action' attribute to send form results to an email address.

Your textarea element will default to 2 rows and 20 columns. To change these values, you must use the following syntax:

<textarea rows="4" cols="40" placeholder="Placeholder text here">
This will result in a textbox with 4 rows and 40 columns. The 'placeholder' attribute ensures that users have an idea of what they should submit in the textbox.

In order to submit a form, you need a submit button. There are two ways you can input a submit button:

<button type="button">Submit Your Form Here!</button>
or
<input type="submit" value="Submit Your Form here!"></input>
Introduction to Styling with CSS

In order to start styling your HTML code with CSS, you must link your CSS document. We addressed this earlier in the Basic HTML Boilerplate section. Your <link> element must go inside your <head> element and must include a 'rel' attribute identifying it as a stylesheet, and an 'href' attribute with the title of the document (including the .css file extension). Including a 'type' attribute identifiying the file as text/css is good practice, but not necessary. The attributes can be written in any order. Notice how this order is different from the one above:

<link href="styles.css" rel="stylesheet" type="text/css"

Once you have linked your CSS stylesheet, you can begin writing CSS code to style your HTML document!

The basic structure of a CSS declaration block is written as follows:

selector {
property: value;
}

Here is an example of how you would style all of your paragraph elements (<p></p>) to have an indent equivalent to 5 spaces in the size of the paragraph font:

p {
text-indent: 5em;
}
Note: one 'em' is equivalent to one space in the font size of the selected element

If you want to style multiple elements, you can do so by separating selectors by a comma. If you wanted to select your first heading and paragraph elements together for example:

h1, p {
font-family: Courier-New;
}

Notice that in the above example, the font family value is separated by a dash. Spaces are not recognized in CSS declaration blocks, and indents must be used where spaces exist in html code or in property or value names.

One way to style several of the same elements without styling all of the elements of that type is to add class attributes to those that you want to style. For example, if you want to style the first, third, and fourth paragraph elements with red text, you can add class values of "red" to each of these paragraphs, then style the class in CSS:

HTML:
<p class="red"></p>
<p></p>
<p class="red"></p>
<p class="red"></p>
<p></p>

CSS:
.red {
color: red;
}

This code will ensure that only the first, third, and fourth paragraph elements have red text. The second and fifth paragraphs will maintain the default black text.

Notice that the way you write a class selector starts with a period: .class

Common CSS Properties

There are over 200 CSS properties you can use to style elements within your page. Here we'll go over some of the more common properties with examples of values:

There are several properties you can choose to change the look of the font in your document. Some of these include 'font-family' to change the font style , 'font-size' to change the size of your font from the default for the font in the selected element, 'font-weight' to change your font from normal weight, to bold or to a specific weight value, or 'color' to change the color of your font within the selected element.

font-family: Courier-New;
font-size: 37px;
font-weight: lighter;
color: #002a63;

In addition to altering the physical appearance of text, you can also alter its position on the page through CSS properties such as:


  • 'text-indent' - which alters the indent of your text

  • 'text-align' - which aligns the text within its parent container (left, right, center, justify, inherit)

  • 'line-height' - which adjusts the spacing between lines of text

  • 'letter-spacing' - which adjusts the spacing between each letter
    • which alter the indent of text,
text-indent: 2em; text-align: center;

Please note that for these two declarations, you must also input a 'display' property value of 'block' for the delcarations to work.

letter-spacing: 3px; line-height: 3em;
line-height: 3em;
CSS Flexbox

CSS Flexbox is a module that allows you to control the sizing, spacing, and positioning of a series of photos within a page and/or container.

To begin styling with flexbox, you must first create a container to hold your images, with an id value unique to that container. A common id for your set of photos is 'gallery', and you can create your container with the <div> element.

HTML:
<div class="gallery">
<img src="" />
<img src="" />
<img src="" />
<img src="" />
<img src="" />
<img src="" />
<img src="" />

After adding in the links to your images, you can begin styling your flexbox. The first things you must do are:


  • Set the container's display value to flex

  • Set your flex-wrap value to define how your items will wrap (wrap, no-wrap (resulting in a scroll bar if the items go past the boundary of the container), wrap-reverse)
  • Set the flex-direction (row or column)

  • Set the dimensions of your container
.gallery {
display: flex;
flex-wrap: wrap;
flex-direction: row;
height: 300px;
width: 95%;
}

The above CSS styling will allow your photos to wrap onto the next row if there is overflow, ensures that the direction of the photos is laid out in rows from left to right (this is the defauly flex-direction value), and gives your container a height of 300px with a width that spans 95% of the page. This responsive width will adjust with the viewer's screen size. The viewport meta tag will ensure that this responsive declaration works on devices of many sizes.

CSS properties that can be applied to the items of your flexbox (here, the .gallery)include 'justify-content' to define where images start to generate and define the spacing in betweeen images, 'align-items' to define which side or axis of the items you will choose to align with each other, and 'gap' to control the spacing between images without altering the space on the outer edges of the gallery/

.gallery img {
align-items: flex-end;
justify-content: space-around;
gap: 22px

The above styling will ensure that the bottoms of your gallery images are lined up, your images are centered in the container and equally spaced from each other (the spacing on the outer edges will not necessarily be the same as the spacing between the images), and there is an equal gap of 22px between each image.
